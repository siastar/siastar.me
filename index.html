ABOUT ME

What I do professionally, is to provide technical solutions to a wide range of possible requests in the IT environment and beyond.

It is a type of activity in which I am quite good, partly because I am a very curious person, deeply interested in technology, but mostly because I read the documentation and the manuals (in addition to Stack Overflow, of course).

I worked extensively in civil engineering, dealing with machinery programming, three-dimensional modeling, prototyping, cad cam design, etc. After that I spent several years with Applecare, both as a remote helpdesk and as a professional trainer, in different countries.


WHY DO I CODE

I like writing code because in the end it is a tool for applying creativity with which it is possible to do almost everything. At least in theory.


WHAT DO I CODE

Narrowing down from "everything" to "something professionally applicable", I am interested in data management, CRUD operations and consequently interactions with databases.

I also use React to build user/admin interfaces and connect them with server and DB (MERN Stack).


PROJECTS
 I work on specific projects such as shopping carts, personal data, address books, archives with access, modification, creation and deletion function, reports, logins, interfaces for users and administrators.

Some of my projects are on gitHub, just in case someone wants to take a look.

Among other things there is also a Simon Game, a bit raw but working, which was the first thing I wrote in Javascript and a RPG game (Hallo Sword!) in a super embryonic state, but writing which I met Node.


WHAT I WANT TO DO
Sure, records and shopping carts are super exciting, but I'd love to work on projects like this:

  • Communication systems between users.
  • Localized services for the internet of things and home automation.
  • OpenGL 3D web app for browsers.
  • Data encryption and security.
  • Mobile application development.
  • Artificial Intelligences.


ABOUT CODING

I do like writing algorithms, but not so much that I want to do it over and over again, so I try to write things that can be easily configured for future reuse.

I gladly use Node and npm, both for the vastness of software available and for the linux-style package management system.

 It would seem that I am a functional programmer, (so they say). I don't know, but if on one hand I am fascinated by objects encapsulation and processes automation, on the other, as long as it is possible, I like to keep things simple.


MY TOOLS

OS

From a programming point of view, I find that using Linux operating systems makes life a lot easier and allows you to concentrate on writing.

Just to mention a couple of reasons, it results very easy to install fundamental development tools such as Apache, MySql, git etc, or still, the remarkable semplicity with which is possible to handle a virtually unlimited number of servers, clients and databases that can be launched and connected each other.


Editor

I started using Emacs while working on an old laptop where Sublime and Atom struggled. After that I never turned back.


Programming Languages

 I write Javascript, html and CSS, in the past I used C++ to create specific management software for small companies.

(PHP in todo list).



Platforms, Frameworks and Databases


Node and the incredible amount of packages available (npm).

React and Bootstrap (Angular in todo list).

Mongo, Mongoose.

Node , Express and http-server.

Creating user interfaces using React, Redux.


AVAILABILITY

I respond to any request, evaluate options and I am available to transfer.

 For contacts and any additional info, I can be reached through the form at the bottom of the page.


Thank You for your interest.

SU DI ME

Ciò che faccio professionalmente è fornire soluzioni tecniche ad un ampio spettro di possibili richieste in ambiente IT e non solo.

E’ un tipo di attività nella quale sono piuttosto bravo, un po’ perchè sono molto curioso e profondamente interessato alle tecnologie, ma più che altro perchè leggo la documentazione ed i manuali (oltre a Stack Overflow, ovviamente).

Ho lavorato a lungo nell'ingegneria civile occupandomi di programmazione macchine, modellazione tridimensionale, prototipazione, disegno cad cam etc. Dopodichè sono stato per diversi anni con Applecare, sia come helpdesk remoto che come trainer professionale, in diversi paesi.


PERCHE' PROGRAMMO

Mi piace scrivere codice perché in ultima analisi è uno strumento di applicazione della creatività con cui è possibile fare quasi tutto. Almeno in teoria.


COSA POSSO PROGRAMMARE

Restringendo il campo da “tutto” a “qualcosa di professionalmente applicabile”, mi interesso di gestione dei dati, operazioni CRUD e di conseguenza delle interazioni con i database, inoltre utilizzo React per costruire interfacce utente (MERN Stack).


PROGETTI

Lavoro su progetti specifici come shopping carts, anagrafiche, rubriche, archivi con funzioni di accesso, modifica, creazione ed eliminazione, reportistica, login, interfacce per utenti ed amministratori.

Alcuni progetti sono su gitHub, nel caso qualcuno volesse dare un'occhiata.

Tra le altre cose c’è anche un Simon Game, un po’ grezzo ma funzionante, che è stata la prima cosa che ho scritto in Javascript ed un RPG game in stato super embrionale, ma scrivendo il quale ho incontrato Node.


QUELLO CHE MI INTERESSA FARE

Certo, anagrafiche e carrelli della spesa sono super eccitanti, tuttavia mi piacerebbe molto lavorare su progetti del tipo:

  • Sistemi di comunicazione tra utenti.
  • Servizi localizzati per l’internet of things e la domotica.
  • Web app 3D openGL per browsers.
  • Crittografia e sicurezza dati.
  • Sviluppo delle applicazioni mobile.
  • Intelligenze Artificiali.


SULLA PROGRAMMAZIONE

Mi piace scrivere algoritmi, ma non così tanto da volerlo fare e rifare all’infinito, quindi cerco di scrivere cose che possano essere facilmente configurabili per riutilizzi futuro.

Utilizzo volentieri Node ed npm, sia per la vastità di software disponibile che per il sistema linux-style di gestione dei pacchetti.

Sembrerebbe che io sia un programmatore funzionale, così dicono. Non saprei, ma se da una parte mi affascinano l’incapsulamento degli oggetti e l’automazione dei processi, finchè è possibile, mi piace tenere le cose semplici.

Editor
